# Identities

User identities on Sonr are powered by decentralized identifiers (DIDs). DIDs are a new type of identifier that is designed to be globally unique and persistent. They are also self-sovereign, meaning that they are controlled by the user and not a centralized authority.

DID documents are a core part of Sonr's decentralized identity system. They provide a way to manage and verify your identity in a secure and decentralized manner. In this guide, we'll explore the different components of a DID document and how to work with them programmatically.

<Note>
  For generating or authenticating users into your application checkout the [Services API](/services).
</Note>


## The DID Document Model

The DID document model contains all the essential information about an identity, such as its public keys, authentication methods, and service endpoints. It provides a means to prove control over the decentralized identifier (DID) associated with the document.

### Properties

<Properties>
  <Property name="context" type="repeated string">
    The JSON-LD context(s) used in the document.
  </Property>
  <Property name="id" type="string">
    The identifier for the DID subject.
  </Property>
  <Property name="controller" type="repeated string">
    The entity/entities controlling the DID subject (optional).
  </Property>
  <Property name="verification_method" type="repeated VerificationMethod">
    The set of verification methods associated with the DID subject (optional).
  </Property>
  <Property name="authentication" type="repeated string">
    The set of authentication methods associated with the DID subject
    (optional).
  </Property>
  <Property name="assertion_method" type="repeated string">
    The set of assertion methods associated with the DID subject (optional).
  </Property>
  <Property name="capability_invocation" type="repeated string">
    The set of capability invocation methods associated with the DID subject
    (optional).
  </Property>
  <Property name="capability_delegation" type="repeated string">
    The set of capability delegation methods associated with the DID subject
    (optional).
  </Property>
  <Property name="key_agreement" type="repeated string">
    The set of key agreement methods associated with the DID subject (optional).
  </Property>
  <Property name="service" type="repeated Service">
    The set of services associated with the DID subject (optional).
  </Property>
  <Property name="also_known_as" type="repeated string">
    The set of identifiers for other personas or identities related to the DID
    subject (optional).
  </Property>
  <Property name="metadata" type="repeated KeyValuePair">
    Additional metadata related to the DID document (optional).
  </Property>
</Properties>

  <Col>

  </Col>
  <Col sticky>

<CodeGroup title="Request" tag="GET" label="/id">

```bash {{ title: 'cURL' }}
curl -G https://highway.build/id
```

```js
import ApiClient from '@sonrhq/ts-client'

const client = new SonrClient(origin)

await client.did.list()
```
